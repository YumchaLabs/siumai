# CPB-050 Journal

Date: 2026-05-16

## Summary

Recorded the compatibility boundary decision for legacy `ContentPart` and closed the workstream.

## Decision

ADR-0008 classifies legacy `ContentPart` as a compatibility carrier, not the canonical request or
response content interface. It should not move to an explicit compatibility namespace immediately
because stable serde payloads, `ChatMessage`, `ChatResponse`, and provider/protocol parity still
depend on it.

The future breaking slice can move or re-export `ContentPart` under `compat` after more main
request serializers and response projections use directional adapters or canonical prompt/output
content parts.
