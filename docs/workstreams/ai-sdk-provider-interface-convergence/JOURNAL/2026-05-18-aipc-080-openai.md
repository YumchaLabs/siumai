# AIPC-080 OpenAI Slice

Date: 2026-05-18
Task: AIPC-080
Status: Done

## Summary

This slice re-audited the `@ai-sdk/openai` package index and filled the remaining public metadata
type gap for OpenAI Responses.

## Landed

- Added AI SDK-style OpenAI Responses metadata envelope types:
  - `OpenaiResponsesProviderMetadata`
  - `OpenaiResponsesReasoningProviderMetadata`
  - `OpenaiResponsesTextProviderMetadata`
  - `OpenaiResponsesCompactionProviderMetadata`
  - `OpenaiResponsesSourceDocumentProviderMetadata`
- Re-exported the envelope types through the OpenAI protocol crate, native OpenAI provider module,
  and `siumai::provider_ext::openai::metadata`.
- Extended the OpenAI public-surface guard to cover the new package names.
- Added a protocol metadata guard proving OpenAI Responses envelopes serialize under `openai`.

## Boundary Note

This slice only adds package-visible metadata type names. It does not move provider-custom replay
events back into the stable stream surface; those remain covered by the AIPC-050/AIPC-060 stream
and bridge boundaries.
