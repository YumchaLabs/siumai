# AIPC-080 Azure Slice

Date: 2026-05-18
Task: AIPC-080
Status: Done

## Summary

This slice re-audited `@ai-sdk/azure` against Siumai's Azure OpenAI package boundary and tightened
the provider-owned metadata surface.

## Landed

- Added AI SDK-style Azure Responses metadata envelope types:
  - `AzureResponsesProviderMetadata`
  - `AzureResponsesReasoningProviderMetadata`
  - `AzureResponsesTextProviderMetadata`
  - `AzureResponsesSourceDocumentProviderMetadata`
- Re-exported the envelope types through `siumai-provider-azure::provider_metadata` and
  `siumai::provider_ext::azure::metadata`.
- Extended public-surface coverage so Azure package imports include the new metadata names.
- Added a provider metadata guard proving Azure Responses envelopes serialize under `azure`, not
  generic `openai`.

## Boundary Note

Azure still reuses OpenAI protocol payload shapes internally, but package-visible metadata is
provider-owned and defaults to `provider_metadata.azure`. The existing explicit override path can
still use `openai` when requested by the caller.
