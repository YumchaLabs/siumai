# AIPC-050 Closeout

Date: 2026-05-18
Task: AIPC-050
Status: Closed

## Summary

AIPC-050 is complete. The slice moved public protocol and gateway stream assertions toward
AI SDK-stable `ChatStreamEvent::Part(ChatStreamPart)` semantics without widening stable parts for
protocol-only replay details.

## Landed Evidence

- OpenAI Responses public feature tests now send tool call/result inputs through stable
  `ChatStreamEvent::Part` values.
- OpenAI Responses has a guard that prevents public feature tests from regressing to
  `Custom("openai:*")` inputs for stable tool stream parts.
- Extras gateway smoke tests now require stable downstream tool stream parts for the
  Anthropic-to-OpenAI Responses route.
- Anthropic and Gemini serializer test sources now guard `Custom` serializer inputs to explicit V3
  compatibility, provider-native, or compatibility tests.

## Boundary Decision

Loose custom-event consumers in extras object/tool-loop/server helpers remain in place. They are
treated as compatibility-boundary readers, not as primary stable provider assertions. AIPC-060 may
add bridge/gateway guards around them if they can hide a stable-part regression, but AIPC-050 should
not remove them as a broad cleanup.

## Gates

The evidence is recorded in `EVIDENCE_AND_GATES.md`. Package-scoped gates passed for OpenAI,
Anthropic, Gemini, and focused extras gateway tests during the AIPC-050 slices.
