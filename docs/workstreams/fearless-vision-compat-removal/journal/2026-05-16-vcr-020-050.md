# VCR-020..050 Journal

Date: 2026-05-16

## Summary

Removed the dedicated vision compatibility surface and closed the workstream.

## Change

- Added a registry source guard that rejects `VisionCapability`, `VisionCapabilityProxy`,
  `Siumai::vision_capability()`, the `LlmClient` vision downcast, and vision-only spec aliases from
  production public sources.
- Removed `siumai-core::traits::VisionCapability`.
- Removed `LlmClient::as_vision_capability(...)`.
- Removed `VisionCapabilityProxy` and `Siumai::vision_capability()`.
- Removed provider/factory vision downcast forwarding from DeepInfra, Fireworks, and TogetherAI.
- Removed `ImageGenRequest`, `ImageResponse`, `VisionRequest`, and `VisionResponse` aliases.
- Updated the beta.7 migration guide, changelog, and compatibility audit.
