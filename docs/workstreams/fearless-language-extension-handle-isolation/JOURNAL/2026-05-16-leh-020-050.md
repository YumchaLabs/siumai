# LEH-020..050 Journal

Date: 2026-05-16

## Summary

Isolated registry language-handle file, skill, and music extension downcasts behind provider-factory
adapters and closed the workstream.

## Change

- Added registry-owned `ClientBackedFileManagementCapability`, `ClientBackedSkillsCapability`, and
  `ClientBackedMusicGenerationCapability` adapters.
- Added explicit `ProviderFactory` extension methods for file, skill, and music surfaces.
- Routed `LanguageModelHandle` extension implementations through those factory methods.
- Tightened registry handle source guards so extension implementations cannot reintroduce
  `compat_language_client_with_ctx(...)` or direct `as_*_capability()` downcasts.
- Updated the architecture compatibility audit.
